Using Git

Version control = Central repo, 1 file name, full history

Distributed vc system, on an open source platform.

Clone

Copy repo to your local machine

Similar to Photocopying the master copy

Commit

Make your changes locally

Commit: A snapshot of your project in time.

changes made to the photocopy are not on the server yet

Push (Sync)

Apply your changes to central repo

Sync your changes to the server

Your Photocopy has now become the new master copy

Why Use GitHub?

Version control = Public central repo

Using GitHub

You have all the features of Git but with the added benefits of GitHub

Fork

Pull
